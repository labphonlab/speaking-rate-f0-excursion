# Speaking rate compresses F0 excursion, not F0 maximum, in spontaneous speech

Reproducible analysis code and manuscript materials for a study of how **local
speaking rate covaries with F0 excursion** in Japanese and English spontaneous
speech (Corpus of Spontaneous Japanese; Buckeye Corpus). Submitted to
*Journal of Phonetics*.

**Headline.** Measured with a **frame-count-robust fixed-landmark F0 excursion**,
faster speech compresses the F0 excursion by ~1.4–1.5 semitones (above the JND) —
a *within-speaker* effect whose mechanism is a **register-raising** of the F0 floor
(F0 minimum rises while the maximum stays flat). F0 *maximum* per vowel carries no
perceptually meaningful rate effect (0.3–0.7 st), and a naïve max−min excursion is
~20% inflated by a frame-count sampling artifact. See `manuscript/` for the paper
and `results/supplement/` for the underlying tables.

```

  manuscript_final.docx        typeset manuscript (Abstract → Conclusion, Tables 1–2, Figs 1–8)
  manuscript_supplementary.docx  Supplementary Table S1 + Supplementary Figures S1–S2

docs/
  manuscript_final/           ★ canonical source text: the finalized sections (Markdown)
  archive/                    superseded development drafts — NOT the final text (see its README)
src/                         extraction + analysis
  diagnose_data.py              (0) pre-extraction health checks
  build_dataset.py              (1) extract F0/intensity/landmark/accent → master CSV
  analysis/01..13_*.R           (2–12) GAMM effect sizes, robustness, language, accent,
                                within/between (Mundlak), intensity, ososagari, mechanism
  *_extract.py, *_check.py      supporting frame-level extractions / QA
scripts/run_all.sh          end-to-end pipeline (steps 0–12)
```

Generated data (`data/`), fitted-model objects, figures scratch, and environment
caches (`.venv/`, `renv/library/`) are **not** version-controlled; they are
regenerated by the pipeline.

## Reproducing the analysis

**1. Dependencies.** Python is managed with [uv](https://docs.astral.sh/uv/)
(`pyproject.toml`, `uv.lock`; F0/intensity via `parselmouth`/Praat). R is managed
with [renv](https://rstudio.github.io/renv/) (`renv.lock`; modelling via `mgcv`).

```bash
uv sync                       # Python env
Rscript -e 'renv::restore()'  # R env
```

**2. Data (obtain separately — not redistributed here).** The corpora are licensed
and must be acquired from their official distributors, then placed or symlinked
under `data/raw/` (these paths are configured in `config/config.yaml`):

| corpus | `data/raw/…` | source |
|---|---|---|
| Corpus of Spontaneous Japanese (monologue) | `csj/` | NINJAL — https://clrd.ninjal.ac.jp/csj/ |
| CSJ (dialogue subset) | `csj_d/` | NINJAL (as above) |
| Buckeye Corpus | `BuckeyeCorpus/` | https://buckeyecorpus.osu.edu/ |

```bash
ln -s /path/to/csj            data/raw/csj
ln -s /path/to/csj_dialogue   data/raw/csj_d
ln -s /path/to/BuckeyeCorpus  data/raw/BuckeyeCorpus
```

The Buckeye label-only subset (`buckeye_data/`) is excluded via
`config.buckeye_exclude_subdirs`.

**3. Run.**

```bash
scripts/run_all.sh            # steps 0–12; ~7 min on a laptop
```

This writes `data/03_processed/rate_f0_master.csv` and regenerates all
`results/supplement/TableS_*.csv` + figures. It reproduces the reported N
(CSJ Monologue 81,687 / Buckeye 274,273 / CSJ Dialogue 21,558 vowel tokens) and
effect sizes. `src/check_docs_numbers.py` cross-checks cited numbers against the
generated tables.
